Step 1 — Math Preparation

Source: MML Ch. 6.4 — Summary Statistics & Independence §6.4

  • Mean (μ): The center of a distribution — computed as the sum of values divided by count. Used to understand the "typical" data point.
  • Variance (σ²): Measures how spread out values are from the mean. High variance = noisier feature.
  • Covariance: Captures how two variables move together. Positive = same direction; Negative = opposite; Zero = no linear relationship.
  • Statistical Independence: Two variables are independent if knowing one gives no information about the other. Key for feature selection decisions.

Days 4–10

Linear Regression

4/10 — First real algorithm, math is foundational
Step 0 — Preview

You will build a model that predicts a continuous value — specifically, house prices — given features like square footage and number of bedrooms. The model finds the best-fit straight line (or hyperplane in multiple dimensions) through data. The output is a number, not a category.

Step 1 — Math Preparation

Sources: MML Ch. 9.1–9.2 (Linear Regression), Ch. 2.2 (Matrices), Ch. 3.8 (Orthogonal Projections)

  • The Linear Model: ŷ = Xθ where X is the feature matrix (n×d), θ is the parameter vector (d×1), and ŷ is the prediction vector. Matrix multiplication computes all predictions at once. §9.1
  • Mean Squared Error (MSE): Loss = (1/n)‖y − Xθ‖². You want to find θ that minimizes this. The norm here comes from the inner product on vector spaces. §9.2
  • Normal Equation (Closed-Form Solution): θ* = (XᵀX)⁻¹Xᵀy — this is maximum likelihood estimation for linear regression, and geometrically it is an orthogonal projection of y onto the column space of X. §3.8, §9.2
  • Matrix Transpose & Inverse: Xᵀ flips rows/columns; (XᵀX)⁻¹ undoes the transformation. Only works if XᵀX is invertible (columns of X are linearly independent). §2.2
Math → Code Mapping
ŷ = Xθ→X @ theta — matrix multiply features by weights for predictions
‖y − ŷ‖²→np.mean((y - y_hat)**2) — the MSE loss to minimize
Normal Equation→np.linalg.inv(X.T @ X) @ X.T @ y — exact solution
XᵀX invertible?→Check np.linalg.matrix_rank(X) — equals number of features

Days 11–18

Logistic Regression & Binary Classification

5/10 — Probability + new loss function, first true classifier
Step 0 — Preview

You will build a model that classifies emails as spam or not spam (or survivors vs. non-survivors on Titanic). Instead of predicting a number, this model outputs a probability between 0 and 1, then applies a threshold to make a decision. This is the foundation of all binary classifiers.

Step 1 — Math Preparation

Sources: MML Ch. 6.6 (Exponential Family, Sigmoid), Ch. 6.3 (Bayes' Theorem), Ch. 8.3 (Parameter Estimation via MLE)

  • Sigmoid Function: σ(z) = 1/(1+e⁻ᶻ) — squashes any real number into (0,1), interpreted as a probability. This is the link function that connects the linear model to a probability output. §6.6
  • Binary Cross-Entropy Loss: L = −[y·log(p) + (1−y)·log(1−p)] — derived from maximizing likelihood of Bernoulli outcomes. This replaces MSE because MSE is non-convex for classification. §8.3
  • Maximum Likelihood Estimation (MLE): Find θ that maximizes the probability of observing the training labels given the features. Minimizing cross-entropy is equivalent to maximizing log-likelihood. §8.3.1
  • Bayes' Theorem Connection: The posterior probability P(class|features) is what we compute — this is Bayesian reasoning applied to classification. §6.3
Math → Code Mapping
σ(Xθ)→1 / (1 + np.exp(-X @ theta)) — forward pass, output = probability
Cross-Entropy→-np.mean(y*np.log(p) + (1-y)*np.log(1-p)) — loss function
Gradient of Loss→X.T @ (p - y) / n — exact gradient (derived from MLE)
Decision Boundary→(p >= 0.5).astype(int) — threshold at 0.5 by default

Days 19–25

Gradient Descent & Optimization

6/10 — The engine behind all of ML, including neural networks
Step 0 — Preview

You will build a pure gradient descent optimizer from scratch, then apply it to both linear and logistic regression instead of using the Normal Equation or sklearn's solver. You will visualize the loss surface and watch the algorithm descend to the minimum. This is the single most important algorithm in all of machine learning.

Step 1 — Math Preparation

Sources: MML Ch. 5.1–5.3 (Vector Calculus), Ch. 7.1 (Gradient Descent)

  • Partial Derivative: ∂L/∂θⱼ — the rate of change of loss with respect to one weight. Tells you which direction to move that specific weight to reduce loss. §5.2
  • Gradient Vector: ∇L = [∂L/∂θ₁, ..., ∂L/∂θd] — stacks all partial derivatives. Points in the direction of steepest ascent. Gradient descent moves in the opposite direction. §5.2, §7.1
  • Gradient Descent Update Rule: θ ← θ − α·∇L — subtract a fraction of the gradient at each step. α (learning rate) controls step size. §7.1
  • Convergence: Gradient descent converges when ‖∇L‖ ≈ 0 (flat region). Convex loss surfaces guarantee a global minimum; non-convex ones do not. §7.1, §7.3
  • Stochastic vs Batch: BGD uses all data for each update; SGD uses one sample; Mini-batch uses a small batch. SGD adds noise that helps escape local minima. §7.1
Math → Code Mapping
∂L/∂θ→grad = X.T @ (y_hat - y) / n — gradient for linear/logistic regression
θ ← θ − α·∇L→theta -= lr * grad — the update step, repeated for n_epochs
Convergence→if np.linalg.norm(grad) < tol: break — stopping criterion
Loss History→losses.append(loss) then plt.plot(losses) — visualize convergence

Days 26–33

Decision Trees & Ensemble Methods

6/10 — No gradient needed, but ensemble intuition is tricky
Step 0 — Preview

You will build a model that predicts loan default risk. A Decision Tree works like a flowchart — it asks a series of yes/no questions about the data to make decisions. Then you'll combine hundreds of trees (Random Forest) to build a more robust, powerful predictor. This is the most used algorithm in industry for tabular data.

Step 1 — Math Preparation

Sources: MML Ch. 6.2 (Discrete Probabilities), Ch. 6.3 (Sum Rule)

  • Entropy: H(S) = −Σ pᵢ·log₂(pᵢ) — measures impurity of a set. A node with 50% class A / 50% class B is maximally impure (high entropy). A pure node (all one class) has entropy 0. §6.2
  • Information Gain: IG = H(parent) − weighted sum of H(children) — the quality of a split. Greedy: at each node, pick the feature/threshold that maximizes IG.
  • Gini Impurity: G = 1 − Σ pᵢ² — alternative to entropy, slightly faster to compute. Both measure how mixed the classes are at a node.
  • Bagging (Bootstrap Aggregating): Train k trees on k different bootstrap samples of data. Average predictions. Reduces variance without increasing bias — the core of Random Forest. §6.3 sum rule context
Math → Code Mapping
Entropy H(S)→-np.sum(p * np.log2(p + 1e-9)) — impurity score at a node
Information Gain→Loop over all features/thresholds, pick highest IG — the splitting criterion
Bootstrap Sample→np.random.choice(n, n, replace=True) — sample indices with replacement
Ensemble Average→np.mean([tree.predict(X) for tree in forest], axis=0)

Days 34–40

Dimensionality Reduction with PCA

7/10 — Eigendecomposition is the deepest math so far
Step 0 — Preview

You will compress a high-dimensional image dataset (e.g., face images with 1000+ pixel features) into 2 dimensions and then visualize it in a 2D scatter plot where similar images cluster together. PCA is the most mathematically elegant algorithm in classical ML — it finds the directions in data where variation is maximized.

Step 1 — Math Preparation

Sources: MML Ch. 10.1–10.4 (PCA), Ch. 4.2 (Eigenvalues/Eigenvectors), Ch. 3.2 (Inner Products)

  • Covariance Matrix: S = (1/n)XᵀX (after centering X). This d×d matrix encodes how all feature pairs co-vary. PCA is a rotation that diagonalizes this matrix. §10.2
  • Eigenvalues & Eigenvectors: Sv = λv — vectors that only scale under the transformation, not rotate. The eigenvector with the largest eigenvalue points in the direction of maximum variance. §4.2
  • Principal Components: The top k eigenvectors form a new basis. Projecting data onto this basis reduces dimensions while retaining maximum variance. §10.2, §10.3
  • Explained Variance Ratio: λᵢ / Σλⱼ — the fraction of total variance captured by component i. Used to decide how many components to keep (e.g., 95% threshold). §10.4
Math → Code Mapping
Covariance Matrix S→S = (X_centered.T @ X_centered) / n
Eigendecomposition→eigenvalues, eigenvectors = np.linalg.eigh(S)
Project to k dims→X_pca = X_centered @ eigenvectors[:, -k:]
Explained Variance→pca.explained_variance_ratio_ — from sklearn PCA

Days 41–47

Support Vector Machines

7/10 — Geometric intuition + constrained optimization
Step 0 — Preview

You will build a classifier that works well on a handwritten digit recognition task — specifically distinguishing two digits — even though the data lives in 784-dimensional space (28×28 pixel images). SVMs find the single "widest road" (maximum margin hyperplane) between two classes, with a clever trick to handle non-linearly-separable data via kernel functions.

Step 1 — Math Preparation

Sources: MML Ch. 12.1–12.4 (SVM), Ch. 3.1–3.3 (Norms, Distances), Ch. 7.2 (Constrained Optimization)

  • Separating Hyperplane: wᵀx + b = 0 defines a decision boundary. The margin is 2/‖w‖ — maximizing the margin means minimizing ‖w‖. §12.1
  • Support Vectors: The data points that lie exactly on the margin boundaries. Only these points determine the hyperplane — all others are irrelevant. §12.2
  • Soft Margin (C parameter): Allows some misclassification. High C = strict margin (possible overfit); low C = wide margin (possible underfit). §12.3, Soft margin §12.3
  • Kernel Trick: k(xᵢ, xⱼ) = φ(xᵢ)ᵀφ(xⱼ) — computes dot products in a high-dimensional space without actually going there. RBF kernel: k(x,z) = exp(−‖x−z‖²/2σ²). §12.4
Math → Code Mapping
Margin = 2/‖w‖→SVM's objective: minimize 0.5 * np.dot(w, w) subject to constraints
Support Vectors→svm.support_vectors_ — inspect which training points "matter"
RBF Kernel→SVC(kernel='rbf', gamma=0.1) — handles non-linear boundaries
C parameter→SVC(C=1.0) — tune with cross-validation

Feedforward Neural Networks

8/10 — Requires mastery of all previous stages
🔴
Neural Network Threshold — Day 48

All prerequisites must be complete before starting this stage. If you have not built gradient descent from scratch (Stage 4), do not proceed. Backpropagation is gradient descent applied recursively through a chain of functions.

Step 0 — Preview

You will build a neural network from scratch that classifies all 10 MNIST handwritten digits (0–9) with >95% accuracy. Unlike logistic regression which applies one linear transformation and a sigmoid, a neural network applies multiple transformations in sequence — each layer extracts more abstract features. You'll implement forward pass, backward pass (backpropagation), and training loop by hand.

Step 1 — Math Preparation

Sources: MML Ch. 5.6 (Backpropagation & Automatic Differentiation), Ch. 5.2–5.3 (Gradients of Vector-Valued Functions), Ch. 7.1 (Gradient Descent)

  • Composition of Functions (Chain Rule): If L = f(g(h(x))), then dL/dx = dL/df · df/dg · dg/dh · dh/dx. Neural networks are precisely such compositions. Backprop applies this recursively from output to input. §5.6
  • Jacobian Matrix: For a vector-valued function f: Rⁿ → Rᵐ, the Jacobian J is an m×n matrix of partial derivatives. The gradient of a vector-valued layer is a Jacobian. §5.3
  • Forward Pass: a⁽ˡ⁾ = σ(W⁽ˡ⁾a⁽ˡ⁻¹⁾ + b⁽ˡ⁾) — each layer applies a weight matrix, adds bias, then applies a nonlinear activation. §5.6
  • Backward Pass (Backpropagation): Propagates the gradient of the loss backwards through each layer using the chain rule. Computes ∂L/∂W and ∂L/∂b for every layer simultaneously. §5.6
  • Activation Functions: ReLU: max(0, z) — sparse activations, does not vanish. Sigmoid: 1/(1+e⁻ᶻ) — used in output layer for binary/probability. Softmax: for multi-class output.
Math → Code Mapping
W⁽ˡ⁾a⁽ˡ⁻¹⁾ + b→Z = W @ A_prev + b — linear transformation at layer l
σ(Z)→A = relu(Z) or softmax(Z) — nonlinear activation
Chain Rule ∂L/∂W→dW = dZ @ A_prev.T / m — gradient w.r.t. weights
Chain Rule ∂L/∂A⁽ˡ⁻¹⁾→dA_prev = W.T @ dZ — propagate gradient to previous layer
Weight Update→W -= lr * dW — same gradient descent as Stage 4, applied to every layer

What You Must Understand
  • Why optimizer.zero_grad() must be called every iteration — PyTorch accumulates gradients by default; forgetting this causes incorrect updates.
  • Why deep networks need nonlinear activations — stacking linear layers (without ReLU/sigmoid) collapses to a single linear transformation, no matter how many layers you add.
  • The train/eval mode distinction: dropout and batch normalization behave differently during training vs. inference. Always call model.eval() before predicting.
  • Backpropagation is not magic — it is the chain rule applied to a computational graph. Every operation (matmul, relu, softmax) has a known gradient formula.

  • Cross-Validation: Partition data into k folds. Train on k−1, evaluate on 1. Rotate k times. Average performance. Gives a reliable estimate of generalization error without touching the test set. §8.2.4
  • Bias-Variance Tradeoff: Total Error = Bias² + Variance + Irreducible Noise. Simple models have high bias (underfit); complex models have high variance (overfit). Cross-validation helps find the sweet spot. §8.2.2
